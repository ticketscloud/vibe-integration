# Vibeapp Integration — Orders Stream

Данная интеграция основана на передаче данных в **Vibeapp** через Kafka-стримы заказов.

_Изменения статусов билетов обрабатываются через **возвраты**._

---

## Overview

Vibeapp получает и обрабатывает следующие типы данных:

- организаторы мероприятий;
- мероприятия;
- заказы с билетами;
- возвраты билетов.

Формат сообщений: **JSON**.

---

## Kafka Topics

### Events

Информация о мероприятиях, организаторов и местах проведения.
Должна быть синхронизирована **до** обработки заказов.

- `events.ts`

---

### Orders

Заказы и купленные билеты.
Основной источник информации о проданных билетах.

- `orders.ts`
- `orders.proto`

---

### Refunds

Возвраты билетов.
Используются для обновления статусов билетов в Vibeapp.

- `refunds.ts`
- `refunds.proto`
